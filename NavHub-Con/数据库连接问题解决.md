# MySQL 连接问题解决方案

## 错误：Access denied for user 'root'@'localhost' (using password: YES)

这个错误表示 MySQL 拒绝了 root 用户的连接，通常是密码不正确。

## 解决方案

### 方案 1：重置 root 密码（推荐）

#### 步骤 1：停止 MySQL 服务

```bash
# Windows（以管理员身份运行）
net stop MySQL80
```

#### 步骤 2：以跳过权限表的方式启动 MySQL

```bash
# 进入 MySQL 安装目录的 bin 文件夹
cd "C:\Program Files\MySQL\MySQL Server 8.0\bin"

# 启动 MySQL（跳过权限验证）
mysqld --console --skip-grant-tables --shared-memory
```

保持这个窗口打开，打开新的命令行窗口继续。

#### 步骤 3：连接 MySQL（无需密码）

```bash
# 新开一个命令行窗口
mysql -u root
```

#### 步骤 4：重置密码

```sql
-- 刷新权限表
FLUSH PRIVILEGES;

-- 重置 root 密码（MySQL 8.0）
ALTER USER 'root'@'localhost' IDENTIFIED BY 'root';
FLUSH PRIVILEGES;

-- 退出
EXIT;
```

#### 步骤 5：重启 MySQL 服务

1. 关闭跳过权限表的 MySQL 窗口（Ctrl+C）
2. 正常启动 MySQL 服务：

```bash
net start MySQL80
```

#### 步骤 6：测试连接

```bash
mysql -u root -p
# 输入密码：root
```

如果成功连接，说明密码已重置。

### 方案 2：修改项目配置使用正确的密码

如果你知道正确的 root 密码，直接修改 `application.yml`：

```yaml
spring:
  datasource:
    username: root
    password: your_actual_password  # 改为你的实际密码
```

### 方案 3：创建新的数据库用户（推荐用于生产环境）

#### 步骤 1：使用 root 登录 MySQL

```bash
mysql -u root -p
# 输入你的 root 密码
```

#### 步骤 2：创建新用户并授权

```sql
-- 创建用户（如果不存在）
CREATE USER IF NOT EXISTS 'navhub'@'localhost' IDENTIFIED BY 'navhub123';

-- 授予权限
GRANT ALL PRIVILEGES ON navhub.* TO 'navhub'@'localhost';

-- 刷新权限
FLUSH PRIVILEGES;

-- 查看用户
SELECT user, host FROM mysql.user WHERE user = 'navhub';
```

#### 步骤 3：修改项目配置

```yaml
spring:
  datasource:
    username: navhub
    password: navhub123
```

---

### 方案 4：检查 MySQL 服务状态

```bash
# 检查 MySQL 服务是否运行
net start | findstr MySQL

# 如果服务未运行，启动它
net start MySQL80
```

## 快速测试连接

### 方法 1：命令行测试

```bash
mysql -u root -p
# 输入密码
```

如果连接成功，会看到：
```
Welcome to the MySQL monitor...
mysql>
```

### 方法 2：使用 MySQL Workbench

1. 打开 MySQL Workbench
2. 创建新连接
3. 输入用户名和密码
4. 测试连接

### 方法 3：修改配置后重启项目

修改 `application.yml` 中的数据库密码后，重启 Spring Boot 项目：

```bash
mvn spring-boot:run
```

查看启动日志，应该看到：
```
HikariPool-1 - Start completed.
```

## 常见问题

### Q1: 忘记 root 密码怎么办？

**A:** 使用方案 1 重置密码。

### Q2: MySQL 服务无法启动？

**A:** 
1. 检查是否有其他 MySQL 实例在运行
2. 检查端口 3306 是否被占用
3. 查看 MySQL 错误日志

### Q3: 连接被拒绝（Connection refused）？

**A:**
1. 确保 MySQL 服务已启动
2. 检查防火墙设置
3. 确认端口号正确（默认 3306）

### Q4: 如何查看 MySQL 错误日志？

**A:**
MySQL 错误日志通常在：
- Windows: `C:\ProgramData\MySQL\MySQL Server 8.0\Data\*.err`
