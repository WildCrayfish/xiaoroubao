# MySQL 8 数据库初始化指南

## 一、MySQL 8 安装后配置

### 1. 启动 MySQL 服务

**Windows:**
```bash
# 方式一：通过服务管理器
# Win + R 输入 services.msc，找到 MySQL80 服务，启动

# 方式二：命令行（以管理员身份运行）
net start MySQL80
```

**验证 MySQL 是否运行:**
```bash
mysql --version
```

### 2. 登录 MySQL

```bash
# 使用 root 用户登录（首次安装可能需要使用临时密码）
mysql -u root -p

# 如果提示输入密码，输入安装时设置的 root 密码
```

### 3. 修改 root 密码（如果需要）

```sql
-- 登录后执行
ALTER USER 'root'@'localhost' IDENTIFIED BY 'root';
FLUSH PRIVILEGES;
```

## 二、创建数据库和表结构

### 方式一：使用 SQL 脚本（推荐）

1. **执行 schema.sql 创建表结构**

```bash
# 在 MySQL 命令行中执行
mysql -u root -p < src/main/resources/db/schema.sql

# 或者在 MySQL 客户端中
source C:/Users/xiaoroubao/Desktop/NavHub-Con/src/main/resources/db/schema.sql
```

2. **执行 data.sql 插入初始数据**

```bash
mysql -u root -p < src/main/resources/db/data.sql
```

### 方式二：手动执行 SQL

1. 打开 MySQL 客户端（如 MySQL Workbench、Navicat 等）
2. 连接到本地 MySQL 服务器
3. 依次执行以下 SQL：

```sql
-- 1. 创建数据库
CREATE DATABASE IF NOT EXISTS navhub DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 2. 使用数据库
USE navhub;

-- 3. 执行 schema.sql 中的所有 CREATE TABLE 语句
-- 4. 执行 data.sql 中的所有 INSERT 语句
```

## 三、配置项目连接数据库

### 修改 application.yml

编辑文件：`src/main/resources/application.yml`

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/navhub?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&zeroDateTimeBehavior=convertToNull
    username: root          # 修改为你的 MySQL 用户名
    password: root          # 修改为你的 MySQL 密码
```

**重要参数说明：**
- `localhost:3306` - MySQL 服务器地址和端口（默认 3306）
- `navhub` - 数据库名称
- `username` - MySQL 用户名（默认 root）
- `password` - MySQL 密码（修改为你设置的密码）

## 六、常见问题

### 问题1：Access denied for user 'root'@'localhost'

**解决方案：**
- 检查用户名和密码是否正确
- 确认 root 用户是否有远程连接权限

```sql
-- 允许 root 用户从 localhost 连接
GRANT ALL PRIVILEGES ON *.* TO 'root'@'localhost' IDENTIFIED BY 'root';
FLUSH PRIVILEGES;
```

### 问题2：Unknown database 'navhub'

**解决方案：**
- 执行 schema.sql 创建数据库
- 或手动创建：`CREATE DATABASE navhub;`

### 问题3：Table doesn't exist

**解决方案：**
- 执行 schema.sql 创建所有表
- 检查是否使用了正确的数据库：`USE navhub;`

### 问题4：时区错误

**解决方案：**
- 确保连接 URL 中包含：`serverTimezone=Asia/Shanghai`
- 或在 MySQL 中设置时区：

```sql
SET time_zone = '+08:00';
```

### 问题5：SSL 连接错误

**解决方案：**
- 在连接 URL 中添加：`useSSL=false`
- 或配置 MySQL SSL 证书
