## 📝 使用说明

### 添加标签
1. 点击"新增工具"或编辑现有工具
2. 在"标签"字段中输入标签，用逗号分隔
3. 例如：`设计, 工具, 免费, UI设计`
4. 保存后标签会自动转换为数组格式

### 搜索标签
1. 在搜索框中输入标签关键词
2. 系统会自动匹配：
   - 工具名称
   - 工具描述
   - 工具标签

### 显示标签
- 工具卡片会自动显示前 3 个标签
- 如果标签超过 3 个，会显示 "+N" 提示
- 标签使用渐变背景，与 NavHub 设计风格一致

## 🔄 后端接口准备

当前前端已完全支持标签功能，后端接口需要支持：

### API 接口要求
1. **创建工具** (`POST /api/tools`)
   - 请求体应包含 `tags` 字段（字符串数组）
   - 示例：
     ```json
     {
       "name": "Figma",
       "description": "...",
       "url": "https://figma.com",
       "icon": "🎨",
       "categoryId": "tools",
       "tags": ["设计工具", "UI设计", "协作"]
     }
     ```

2. **更新工具** (`PUT /api/tools/{id}`)
   - 支持更新 `tags` 字段

3. **获取工具** (`GET /api/tools`)
   - 返回的工具对象应包含 `tags` 字段

4. **搜索功能** (`GET /api/tools?search=关键词`)
   - 后端搜索应支持标签匹配
   - 建议在 SQL 查询中包含标签字段的搜索

### 数据库字段
- 建议在 `tool` 表中添加 `tags` 字段
- 类型：`JSON` 或 `TEXT`（存储 JSON 数组）
- 示例：`["设计工具", "UI设计", "协作"]`
