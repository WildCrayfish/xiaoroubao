## ❌ 常见问题排查

### 问题 1：登录失败，提示"网络错误"

**可能原因：**
- 后端服务未启动
- 后端端口不是 8080
- 防火墙阻止连接

**解决方法：**
1. 检查后端是否运行：访问 `http://localhost:8080/api/auth/login`
2. 检查后端日志是否有错误
3. 确认后端端口是 8080

### 问题 2：登录失败，提示"用户名或密码错误"

**可能原因：**
- 用户名或密码输入错误
- 数据库中用户不存在
- 密码加密方式不匹配

**解决方法：**
1. 确认用户名：`admin`，密码：`admin123`
2. 检查数据库：
   ```sql
   SELECT * FROM navhub.user WHERE username = 'admin';
   ```
3. 如果用户不存在，执行 `init.sql` 初始化数据

### 问题 3：登录成功但无法获取用户信息

**可能原因：**
- Token 格式错误
- Token 未正确保存
- 后端 Token 验证失败

**解决方法：**
1. 检查 localStorage 中是否有 token
2. 检查请求头中是否携带了 Token
3. 查看后端日志，确认 Token 验证是否通过

### 问题 4：CORS 跨域错误

**可能原因：**
- 后端跨域配置不正确
- 前端请求的 URL 不正确

**解决方法：**
1. 确认后端 `CorsConfig.java` 已配置
2. 确认前端请求的 URL 是 `http://localhost:8080/api`

### 问题 5：401 未授权错误

**可能原因：**
- Token 过期
- Token 格式错误
- Token 未携带

**解决方法：**
1. 重新登录获取新 Token
2. 检查 Token 是否正确保存
3. 检查请求头格式：`Authorization: Bearer <token>`
