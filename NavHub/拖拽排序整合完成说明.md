# 拖拽排序功能整合完成说明

## ✅ 已完成的工作

### 1. 安装依赖
- `@dnd-kit/core` - 拖拽核心库
- `@dnd-kit/sortable` - 排序功能
- `@dnd-kit/utilities` - 工具函数

### 2. 创建 SortableTagCard 组件
- **文件**: `src/app/components/sortable-tag-card.tsx`
- **功能**: 
  - 基于 `useSortable` 包装 TagCard
  - 管理员模式下显示拖拽手柄（GripVertical 图标）
  - 拖拽手柄位于卡片左上角
  - 保持 NavHub 的玻璃态 UI 风格
- **样式**: 拖拽手柄使用白色半透明背景，悬停时显示 indigo 色

### 3. API 层更新
- **toolApi.reorder(orderedTools: Tool[])**: 批量更新工具排序
- **mockApi.reorderTools(orderedTools: Tool[])**: Mock 数据排序更新
- 支持 Mock 模式和真实 API 模式

### 4. useTools Hook 更新
- 新增 `reorderTools(orderedTools: Tool[])` 方法
- 调用 API 更新排序并同步本地状态

### 5. Mock 数据更新
- 为所有工具添加 `sort` 字段（0-7）
- `getTools` 返回结果按 `sort` 字段排序
- `reorderTools` 更新 mockTools 中的 sort 值

### 6. Home 页面集成
- **DndContext**: 包裹拖拽区域
- **SortableContext**: 使用 `rectSortingStrategy` 适配网格布局
- **PointerSensor**: 配置 8px 激活距离，避免误触
- **handleDragEnd**: 使用 `arrayMove` 处理排序逻辑
- **条件渲染**: 仅管理员可见拖拽功能，普通用户使用普通 TagCard

## 🎨 使用方式

### 管理员模式
1. 使用 admin/admin123 登录（Mock 模式）
2. 工具卡片左上角显示 **拖拽手柄**（六条横线图标）
3. 按住手柄拖拽可调整工具顺序
4. 松开后自动保存新顺序

### 普通用户
- 不显示拖拽手柄
- 使用普通 TagCard，无拖拽功能

## 📋 技术实现细节

### 拖拽策略
- **rectSortingStrategy**: 适配网格布局（grid-cols-1 sm:grid-cols-2 lg:grid-cols-3...）
- **closestCenter**: 碰撞检测算法

### 排序逻辑
- 拖拽结束后通过 `arrayMove` 计算新顺序
- 调用 `reorderTools` 更新 API 和本地状态
- Mock 中直接更新 mockTools 的 sort 字段

### 分类/搜索场景
- 筛选后的列表内可拖拽排序
- 仅更新当前可见工具的 sort 值
- 保持与其他分类工具的相对顺序

## 🔄 后端接口准备

当前前端已支持拖拽排序，后端需实现：

### 接口要求
1. **批量更新排序** (`PUT /api/tools/reorder`)
   - 请求体: `{ tools: Tool[] }`

### 数据库
- `tool` 表需有 `sort` 字段（INT 或 BIGINT）
- 数值越小排序越靠前

## 📝 注意事项

- 拖拽手柄仅在**管理员登录**时显示
- 移动 8px 后才触发拖拽，避免与点击冲突
- 拖拽时卡片半透明（opacity: 0.5）提供视觉反馈
